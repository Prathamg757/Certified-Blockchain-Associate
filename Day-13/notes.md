# Day 13 - Decentralized Applications (DApps)

## Overview

This module introduces Decentralized Applications (DApps), Smart Contracts, Remix IDE, Tokens, Popular Ethereum DApps, and Smart Contract Programming Languages.

---

# 1. Decentralized Applications (DApps)

## Definition

A DApp (Decentralized Application) is software that interacts with a blockchain and manages the state of network participants without relying on a central authority.

---

## DApp Architecture

```
User

↓

Front End

↓

Web3

↓

Smart Contract

↓

Ethereum Virtual Machine (EVM)

↓

Operating System
```

---

## Components

### Front End

- User Interface (UI)
- Allows users to interact with the application

### Web3

- APIs and Libraries connecting the UI with Ethereum
- Sends transactions to the blockchain

### Smart Contract

- Contains business logic
- Executes predefined rules automatically

### Ethereum Virtual Machine (EVM)

- Executes Smart Contracts

### Operating System

- Platform where the EVM runs

---

# 2. Smart Contracts

## Definition

A Smart Contract is a self-executing program stored on the Ethereum blockchain that automatically executes when predefined conditions are satisfied.

---

## Features

- Transparent
- Tamper-proof
- Fraud Reduction
- Zero Third-Party Intervention
- Cost Efficient
- Permanent Record Keeping
- Anonymity
- Self-executing
- Self-verifying
- Immutable

---

## Smart Contract Flow

```
User

↓

Smart Contract

↓

Condition Check

↓

Automatic Execution

↓

Blockchain Updated
```

# 4. Tokens

## Definition

Crypto Tokens are digital representations of assets or value stored and transferred on a blockchain.

---

## Main Features

- Digital Asset Representation
- Represents Tangible & Intangible Assets
- Blockchain Ownership
- Smart Contract Transfer
- Programmable

---

## Token Characteristics

- Liquidity
- Divisibility
- Tradability
- Immutable Proof of Ownership

---

## Types of Tokens

### A) Equity Token

- Represents ownership
- Similar to company shares

---

### B) Utility Token

- Platform-specific token
- Used as service or coupon token

---

### C) Security Token

- Regulated financial asset
- Provides additional security and compliance

---

# 6. Smart Contract Programming Languages

## Solidity

### Definition

A high-level object-oriented programming language proposed by Gavin Wood in 2014 for Ethereum Smart Contract development.

### Features

- JavaScript-like syntax
- Object-Oriented
- Statically Typed
- Supports Inheritance
- Supports Libraries
- Supports Mapping & Struct

---

## Vyper

### Definition

A Python syntax-based Smart Contract language focused on simplicity and security.

### Features

- Python-like syntax
- Strongly Typed
- Bounds & Overflow Checking
- Signed & Fixed-point Numbers
- Simple Compiler
- Highly Auditable

# Key Takeaways

- DApps eliminate centralized control by running on blockchain networks.
- Smart Contracts automatically execute business logic without intermediaries.
- Remix IDE provides an easy environment for developing Ethereum Smart Contracts.
- Tokens represent digital assets and can be Equity, Utility, or Security Tokens.
- Solidity and Vyper are the primary programming languages used for Ethereum Smart Contract development.

# Summary

Decentralized Applications (DApps) combine Front End interfaces, Web3 libraries, Smart Contracts, and the Ethereum Virtual Machine to provide trustless applications. Smart Contracts automate processes, Tokens represent digital assets, and tools like Remix IDE simplify development. Solidity and Vyper enable developers to build secure blockchain applications.
