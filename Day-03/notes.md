# 1. Introduction

Blockchain securely stores and verifies information using **Hashing** and **Merkle Trees**.

It answers two important questions:

- How is data stored securely?
- How is data protected from modification?

## Merkle Root

A **Merkle Root** (Root Hash) is generated using a Merkle Tree.

It acts as the **digital fingerprint** of all transactions in a block.

# Summary

Blockchain achieves **immutability** using **Merkle Trees, Merkle Roots, Hashing, and Previous Hashes**. Every block is linked to the previous one, making unauthorized modifications immediately detectable. A decentralized blockchain network with multiple nodes ensures transparency, security, and fault tolerance.
